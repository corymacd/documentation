To create a Redpanda cluster in the Dedicated Cloud, log in to the Redpanda Console, then follow the steps for creating a cluster, choosing a cloud provider, and creating a network.


## Log in to Redpanda Console

These instructions explain how to log in to the Redpanda Console, add a namespace, and select Dedicated Cloud for deploying your Redpanda cluster.

1. Go to the Redpanda cloud URL and click **Login**.
The **Enter Organization** page opens.
2. Type your organization name and click **Continue**.
    The Home page appears.
3. Click **Add namespace**.
    Namespaces help you organize clusters by owner, team, or project for example
4. Type a name for your namespace.
5. Scroll to your namespace (namespaces are listed in alphabetical order) and click **+ Add Cluster**.
6. Select **Dedicated**.

## Create a cluster

After you finish logging in and you select Dedicated Cloud, create a Redpanda cluster:

1. Click **Create Dedicated cluster**.
    The **Cluster settings** page appears.
2. (Optional) In **Cluster name**, type a name to replace the default name.
3. Select throughput. The choices are 1 MBps, 5 MBps, 10 MBps, and 20 MBps.
4. Open the Redpanda version menu and select a version.
5. (Optional) Click the **Enable connectors** slider to turn on this feature.
6. Click **Next**.
    The Cloud settings page appears.
7. For **Provider**, select AWS.
8. Open the **Region** menu and select a region.
    The choices are eu-west-3 and us-west-2. Choose the region that is geographically closest to you.
9. Open the **Zones** menu and select three availability zones, one at a time, for your region.
10. Click **Next**.
    The **Networking** page appears.
11. (Optional) In **Network name**, type a name to replace the default name.
12. In **IP Range**, type a range of IP addresses using CIDR notation to represent your network.
    For Class A, use 10.0.0.0/<number of bits in the netmask>. For Class B, use 172.16.0.0/<number of bits in the netmask>. For Class C, use 192.168.0.0/<number of bits in the netmask>. The default network IP range is 10.0.0.0/16.
13. Click **Create - (network, then cluster)**.

The system displays the message “Cluster creation can take a while. This page will refresh when cluster creation is complete.”
